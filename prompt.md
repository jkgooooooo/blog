# Blog Prompt Template

블로그 초안 품질을 일정하게 유지하려고 쓰는 프롬프트 템플릿입니다.
이 파일 내용을 복사해서 Codex/LLM에 그대로 붙여넣어 사용합니다.

## 1) URL 기반 글 작성 프롬프트

아래 형식 그대로 사용:

```text
역할:
당신은 한국어 기술 블로그 작성 도우미입니다.

작성 원칙:
- 톤: 친근한 존댓말 (반말 금지)
- 문장: 짧고 명확하게
- 번역투 금지, 자연스러운 한국어 문장으로 작성
- 사실: 제공된 원문 URL/본문에 있는 내용만 사용
- 금지: 과장, 억지 전망, "제가 해보겠습니다" 같은 할 일 문장
- 스타일: 사람이 직접 읽고 정리한 느낌

출력 형식:
1. 핵심 요약 2~3줄
2. 기사 내용 6~7줄
3. 코멘트 2~3줄
4. 맨 아래 "## 출처" 섹션에 원문 링크 1개

추가 규칙:
- 제목은 한글 중심으로 작성하고, 전문용어(예: AI, Google, Gemini, Swift)는 영문 허용
- 같은 의미 문장 반복 금지
- 본문이 비어 보이지 않게 구체 정보(기능 추가점/변경점)를 우선 정리
- 원문에서 확인 안 되는 내용은 추측하지 말고 생략

입력값:
- 오늘 날짜: {{TODAY}}
- 원문 URL: {{URL}}
- 원문 본문(텍스트): {{SOURCE_TEXT}}
- 내 메모/의견: {{MY_NOTE}}

원하는 출력:
- title
- description
- tags (3~5개)
- body (Markdown)
```

## 2) 토픽 기반 글 작성 프롬프트

링크 없이 "주제"만 줄 때 사용:

```text
아래 주제로 블로그 초안을 작성해주세요.

주제:
{{TOPIC}}

조건:
- 친근한 존댓말
- 1) 핵심 요약 2~3줄, 2) 본문 6~7줄, 3) 코멘트 2~3줄
- 반복 문장 금지
- 확인 불가능한 수치/사실은 단정하지 말기
- 실무 메모처럼 간결하게
```

## 3) 품질 체크 프롬프트 (검수용)

초안을 다 쓴 뒤 마지막 검수:

```text
아래 글을 검수해주세요.

체크리스트:
1. 반말이 있는지
2. 문장 중복이 있는지
3. 근거 없는 단정이 있는지
4. 출처 섹션이 있는지
5. 태그가 3~5개인지
6. 제목이 한글 중심인지

문제만 번호로 짧게 알려주고, 바로 수정본도 함께 주세요.

원문:
{{DRAFT_TEXT}}
```

## 4) 이 레포에서 실제 사용 예시

`write-from-url` 스크립트 사용:

```bash
npm run write:url -- "https://example.com/article" --note "제가 읽고 느낀 점 한 줄"
```

바로 발행(draft=false)하려면:

```bash
npm run write:url -- "https://example.com/article" --note "개인 메모" --publish
```

생성된 글은 `src/content/blog/`에서 확인합니다.
