Anthropic이 Claude Opus 4.6을 공개했습니다.  
이번 발표를 보면 단순히 “더 똑똑해졌다”보다, Claude Code처럼 실제로 오래 붙잡고 쓰는 작업에서 얼마나 덜 흔들리느냐를 강하게 밀고 있다는 느낌이 들었습니다.

특히 코딩, 컴퓨터 사용, 툴 사용, 검색, 금융 분석 같은 실무형 작업을 한 번에 묶어 설명한 점이 인상적이었습니다.  
개인적으로는 여기서 제일 눈에 들어온 건 Opus 계열 처음으로 1M context window를 베타로 넣었다는 부분이었습니다.

## 핵심 요약

- Claude Opus 4.6은 Anthropic이 “가장 똑똑한 모델”을 한 단계 올린 버전으로 소개했습니다.
- 발표 포인트는 에이전트형 코딩, 컴퓨터 사용, 툴 사용, 검색, 금융 분석 같은 긴 업무 흐름 전반입니다.
- 코딩 쪽에서는 더 신중하게 계획하고, 더 오래 작업을 유지하고, 큰 코드베이스에서 더 안정적으로 움직이는 점을 강조했습니다.
- Opus 클래스 모델 최초로 1M token context window를 베타로 지원합니다.

## 기사 내용

1. Anthropic 설명 기준으로 Opus 4.6은 이전 모델보다 코딩 능력이 좋아졌고, 특히 에이전트형 작업을 더 오래 안정적으로 유지할 수 있도록 개선됐습니다.
2. 발표문에서는 더 신중하게 계획하고, 큰 코드베이스에서도 더 안정적으로 움직이고, 코드 리뷰와 디버깅에서도 자기 실수를 더 잘 잡는다고 설명합니다.
3. Opus 4.6은 코딩 외에도 금융 분석, 리서치, 문서·스프레드시트·프레젠테이션 작업까지 일상적인 지식 노동 범위 전반을 같이 밀고 있습니다.
4. Anthropic은 이 모델이 에이전트형 코딩 평가, 깊은 다단계 검색, 경제적 가치가 있는 지식 업무 평가에서 강한 성능을 보인다고 밝혔습니다.
5. 컴퓨터 사용과 검색을 묶은 흐름도 계속 강화하고 있고, 발표문 전체 톤도 “단발성 질문 응답”보다 “긴 작업을 끝까지 밀어주는 모델”에 가깝습니다.
6. Claude Code 쪽에서는 실제 내부 엔지니어들이 매일 테스트하고 있으며, 어려운 구간에 더 집중하고 쉬운 구간은 빠르게 넘기며 긴 세션에서도 생산성을 유지한다고 설명했습니다.
7. 외부 파트너 코멘트도 비슷했습니다. Notion, GitHub, Replit, Asana, Cognition, Windsurf 같은 곳 모두 복잡한 멀티스텝 작업, 계획 수립, 디버깅, 큰 코드베이스 탐색에서 좋아졌다는 식으로 평가했습니다.
8. 배포 범위는 발표문 기준으로 Claude, API, 주요 클라우드 플랫폼에서 바로 제공되며, 개발자는 Claude API와 Claude Code에서 사용할 수 있다고 안내했습니다.

## 코멘트

- 이번 업데이트는 “성능표 한 줄 더 올라감”보다, 실제로 Claude Code에 길게 일을 맡길 때 중간에 덜 흔들리는지가 핵심인 것 같습니다.
- 특히 1M context beta는 긴 코드베이스나 긴 작업 세션을 자주 다루는 사람 입장에서는 체감 포인트가 확실할 것 같습니다.
- 다만 이런 모델은 결국 벤치마크보다 실사용에서 얼마나 덜 새고, 얼마나 끝까지 밀어주는지가 중요해서, 실제 코딩 워크플로우에 넣어봤을 때 느낌을 더 봐야 할 것 같네요.
